# Campaign Structure — Project vs Theme Field

## When to create a Notion Project (in Projects DB)
Create a full Project entry when ANY of these are true:
- 5 or more content pieces in the campaign
- Paid budget involved (Meta ads, influencer, etc.)
- Campaign runs 2+ weeks
- Multiple team members involved
- Campaign has a specific launch date or event

A Project entry should include:
- Project name
- Brand
- Goal
- Timeline (start → end)
- Budget (if applicable)
- Content Calendar relation (link all CC entries)
- Status

## When to use Campaign/Theme field only (no Project)
Use Theme field on Content Calendar entries only when:
- Fewer than 4 pieces, all organic
- Quick turnaround (under 1 week)
- Single creator involved
- No paid amplification planned

## Campaign/Theme field naming
- Must be IDENTICAL across all pieces in the same campaign
- Format: [Year] [Campaign Name] e.g. "2026 Tennis Elbow Prevention"
- For seasonal themes: "Q2 2026 Spring Injury Season"
- For product launches: "2026 Heel Pad XC Launch"

## CC-ID assignment within campaigns
- Assign sequentially: CC-001, CC-002, CC-003...
- CC-IDs are campaign-scoped (each campaign starts at CC-001)
- If campaign grows beyond CC-010, create Project entry retroactively

## Archiving
- Completed campaigns: set all entries to "Published" status
- Keep Campaign/Theme field — used for historical research
- Do not delete entries — archive in Notion instead
