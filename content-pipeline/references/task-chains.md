# Task Chains — Per Format

These chains define the production task sequence for each content format.
Use these when creating tasks in Notion All Tasks DB during Phase 6.

## Format: Instagram Reel / TikTok / YouTube Short
1. [CC-ID] Reel – Write Script
2. [CC-ID] Reel – Generate TTS (Kie.AI)
3. [CC-ID] Reel – Generate Images / Video Clips
4. [CC-ID] Reel – Edit in Canva
5. [CC-ID] Reel – Write Caption
6. [CC-ID] Reel – In Review
7. [CC-ID] Reel – Schedule / Post

## Format: YouTube Long-Form
1. [CC-ID] YT Long – Write Script
2. [CC-ID] YT Long – Generate TTS (Kie.AI)
3. [CC-ID] YT Long – Generate Images / B-Roll
4. [CC-ID] YT Long – Edit in Canva / DaVinci
5. [CC-ID] YT Long – Create Thumbnail
6. [CC-ID] YT Long – Write Description + Tags
7. [CC-ID] YT Long – In Review
8. [CC-ID] YT Long – Upload / Schedule

## Format: Static Image (Feed Post)
1. [CC-ID] Static – Design Brief
2. [CC-ID] Static – Generate Image(s)
3. [CC-ID] Static – Canva Composite
4. [CC-ID] Static – Write Caption
5. [CC-ID] Static – In Review
6. [CC-ID] Static – Schedule / Post

## Format: Email Drip Sequence
1. [CC-ID] Drip – Map Sequence (N emails)
2. [CC-ID] Drip – Write All Drafts
3. [CC-ID] Drip – In Review
4. [CC-ID] Drip – Build in Mailchimp
5. [CC-ID] Drip – Test All Sends
6. [CC-ID] Drip – Approve + Activate

## Dependency rules
- TTS generation depends on Script approval
- Image generation depends on Script/Brief approval
- Canva edit depends on all assets (TTS + images) being generated
- Caption writing can happen in parallel with editing
- Review always final before Schedule/Post
